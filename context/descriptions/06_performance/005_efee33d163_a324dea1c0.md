## Changes

The three "High-Impact Formulation Reductions" items from `context/descriptions/todo.md`, landed as one bundled commit. Plan: `~/.claude/plans/lets-do-all-3-polished-dragon.md`.

1. **CoefficientColumn (Scalar | Dense).** Replaced `vector<vector<double>> row_coefficients` and `vector<double> rhs_values` on `EvaluatedConstraint` (and the parallel `BilinearTerm`, `QuadraticGroup`, `SolverInput::objective_coefficients`, `quadratic_inner_coefficients`, `BilinearObjectiveTerm`) with `CoefficientColumn` — a tagged value that stores either a scalar broadcast or a dense `vector<double>`, with lazy Scalar→Dense promotion on mutation. Broadcast-construction sites (hard MIN/MAX indicator coeff, McCormick L1–L4 constants, ABS upper-bound `±2M`, constant-RHS aggregates, NE indicator when no WHEN/PER) now call `MakeScalar(K, num_rows)` instead of allocating `num_rows × 8` bytes. The model builder reads via `Get(row)` and short-circuits the aggregate RHS uniformity check via `IsUniform()` when the column was never promoted.

2. **CSR group metadata on `EvaluatedConstraint` and `SolverInput`.** Added `group_offsets` (`size = num_groups + 1`) and `group_row_ids` (flattened CSR), plus `objective_group_offsets` / `objective_group_row_ids`. New `BuildGroupCSR` helper in `ilp_model_builder.cpp` (declared in `ilp_model.hpp`). The four `vector<vector<idx_t>> group_rows(num_groups); for (row...) group_rows[gid].push_back(row)` reconstructions in the linear builder, the quadratic builder, the deferred-NE expansion, and the PER objective MIN/MAX path now share one CSR built once per constraint/objective.

3. **Active-row indicator allocation for hard MIN/MAX.** The flat MIN/MAX objective (PATH A) and PER inner MIN/MAX (PATH B) used to allocate one global indicator binary per row of the input — even rows that the objective WHEN mask filtered out, even rows with all-zero coefficients across terms. Now PATH A computes `active_rows` (passes WHEN AND has at least one nonzero coefficient) and allocates `active_rows.size()` indicators; the `SUM(y) >= 1` constraint sums only over those. PATH B reuses the CSR's `obj_flat_rows` (which already enumerates only WHEN-active grouped rows) and uses each row's CSR position as its active index. Easy paths skip vacuous linking constraints over inactive rows. An empty active set throws `InvalidInputException` instead of an unbounded MIP — `RejectEmptyAggregate` (line 2734) still catches the WHEN-filters-everything case earlier, so this is a defense-in-depth guard for the all-zero-coefficient corner.

## Hypothesis

- Item 1: shrinks per-constraint memory and cuts the dense init loops on broadcast sites. Most visible on queries with many MIN/MAX/NE/ABS expansions or McCormick bilinear linkings (each adds N broadcast columns of length `num_rows`).
- Item 2: removes redundant `O(num_rows)` reconstruction passes per constraint/objective with PER. Visible on PER queries (Q4, Q7).
- Item 3: shrinks the MIP variable count and constraint row count for hard MIN/MAX with WHEN filters. Should help the solver, not just the model build, when filtering is significant.

## Reading the results

- **Model build (Item 1+2 territory):** -2.2% aggregate, -3 to -8% on the model-heavy queries (Q1, Q2 medium, Q5, Q6, Q7). Every single query is flat-or-better on `model_construction_ms`. This is exactly the sub-second-per-query stage the optimizations were aimed at.
- **Memory:** consistent 1–6% RSS reduction on most queries; -64 MB on Q1 large and -98 MB on Q2 large. The compact-column wins scale with `num_constraints × num_rows` for queries that emit many broadcast LHS terms (Q2's ABS expansion, Q1's per-row M-coefficient blocks).
- **Q4 -25.6% wall-time on large:** model time was already flat — this is the Item 3 active-row indicator allocation shrinking the MIP that gets handed to Gurobi. Q4 is `nested_per_objective` (PER MIN/MAX inner), which previously allocated `num_rows` indicators per occurrence; now allocates only the WHEN-active grouped rows.
- **Q9 +25% wall-time on large is solver noise, not a regression.** `model_construction_ms` is 13.56 → 13.68ms (unchanged within rounding). Q9 is a pure quadratic solve where Gurobi's branch-and-bound dominates a 22s wall time; we already saw ±10s swings on this query in the previous perf log (entry 004 noted 13–27s range).
- Q4 medium showed +1.5% on model time — within run-to-run noise (the medium scale's PER groups are small enough that the CSR build overhead and the `for k = offsets[g]; ...` loop overhead show up before the saved reconstruction pays off).

## What this leaves on the table

- **Item 1 sparse-masked variant**, deferred. The NE per-row indicator path (`physical_decide.cpp` ~line 3066) still allocates a Dense column with mostly zeros and `-M` on the ~10% of active rows. A SparseMasked variant would cut both bytes and the builder's iteration cost for that one path. Skipped here because the post-fill mutation patterns (filter zero-mask, AVG in-place scaling) on the broader column population would force aggressive promotion-back-to-Dense and eat the win.
- **Streaming aggregates directly into accumulators during evaluation** (the original Item 2 framing) is still not done — we hoisted the group CSR but still materialize per-row coefficients first. The MIN/MAX, NE, ABS, and McCormick expansion paths read `row_coefficients[t][row]` after the dense build, so eliminating the intermediate would require restructuring all of those derivations.
- **Per-row zero-coefficient filtering in PER objective easy paths** (PATH B inner easy, lines ~3631–3650) — could shave more constraint rows when `saved_obj_coefficients[t][row]` is zero across all terms within a non-empty group. Not measurably important on the current bench.
