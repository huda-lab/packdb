**Commit**: `9c3a53fb62` (`decide: four model-building speed optimizations`)
**Baseline**: `d534f8b8ed`
**Benchmark JSONs**: `benchmark/decide/results/9c3a53fb62.json` vs `benchmark/decide/results/d534f8b8ed.json`
**Suite**: `benchmark/decide/run_benchmarks.py`, sizes `medium` + `large`, 3 iterations each.

## Changes

Five distinct hot-path edits in `physical_decide.cpp`, `ilp_model_builder.cpp`, and `deterministic_naive.cpp`:

1. **Variable-bound absorption.** `TraverseBoundsConstraints` now runs in the gstate constructor and records every absorbed `BOUND_COMPARISON` pointer in `gstate.absorbed_bound_exprs`. `AnalyzeConstraint` skips those leaves so simple `x <= K` / `x >= L` / `x = K` / `x BETWEEN ...` declarations no longer also emit `num_rows` redundant per-row model rows. Strict `<` / `>` on REAL variables remain unabsorbed so `ApplyComparisonSense` can still reject them with a clear error.
2. **Row-scoped aggregate fast path.** `CanUseRowScopedFastPath()` selects a hash-map-free emission path when every term references a row-scoped decide variable and no decide variable repeats across terms. Coefficients are pushed directly into `constr.indices` / `constr.coefficients` with a precomputed reserve; entity-scoped vars or duplicated decide vars still go through the `unordered_map` accumulator. Applied to both ungrouped and grouped aggregate branches.
3. **Vector reservations.** `model.constraints` is reserved using a one-pass estimate (aggregates → 1 or `num_groups`, per-row → `num_rows`). Per-row constraints reserve `indices` / `coefficients` by active term count. `deterministic_naive.cpp` precomputes total nnz across `model.constraints` and reserves `a_rows` / `a_cols` / `a_vals` / `row_lower` / `row_upper` in one shot.
4. **Tautology dropping.** A new `PushNormalizedConstraint()` helper gates the three linear emission sites. Empty-LHS tautologies (`0 <= k>=0`, `0 >= k<=0`, `0 = 0`) are dropped; violated empty-LHS rows throw `InvalidInputException` with an infeasibility message before ever reaching the solver.
5. **Deferred `saved_obj_coefficients` copy.** The `vector<vector<double>>` deep copy is gated by `need_saved_obj`, only taken for the PER-nested-aggregate path or the flat-aggregate objective path. Plain linear objectives no longer pay for a copy of `num_terms × num_rows` doubles.

## Hypothesis

The previous benchmarks showed model-construction time growing linearly with `num_rows × num_constraints`, and a non-trivial chunk of that time came from emitting per-row model rows for constraints that the column-bounds machinery had already absorbed. Most generated linear constraints in the benchmark suite are pure row-scoped aggregates with no entity vars, which made the unconditional `unordered_map` accumulator unnecessary for the common case. The remaining items (reservations, tautology drop, deferred copy) are pure overhead reduction with no semantic change.

### Apparent regressions (medium size only, both solver-bound)

- **Q1 medium**: wall +65%, solver +79%, model −65%.
- **Q9 medium**: wall +5.8%, solver +8.6%, model −46%.

Q1 large and Q9 large are firmly in the win column on the same change. Both queries are ≥97% solver-bound, and Gurobi MIP wall time has substantial run-to-run variance unless `Seed` and threading are pinned. Treating these as solver-time noise rather than a regression caused by the changes; revisit if they reproduce on a re-run with a fixed seed.
