Known Limitations

As a research prototype, PackDB has several constraints that will be addressed in future releases:

Linearity Requirement

Only Linear Programming (LP) and Integer Linear Programming (ILP) are supported. Cannot express quadratic objectives, variable interactions, or non-linear functions.

Scalability Constraints

Exact solving works well for ~1K-10K rows. Performance degrades significantly beyond this range due to NP-Hard nature of integer programming.

Subquery Restrictions

Only uncorrelated scalar subqueries are supported. Correlated subqueries that reference outer query columns are not yet implemented.

Variable Types

No support for continuous (real) decision variables. Only BINARY {0, 1} and non-negative INTEGER {0, 1, 2, ...} variables are available.

Memory Requirements

All candidate rows are materialized in memory. Very large candidate sets may exceed available RAM. Use WHERE clauses to filter aggressively.
