Nurse Scheduling

Advanced
Table-Scoped Variables JOIN Optimization PER Clause
THE PROBLEM

A hospital needs to select nurses to cover shifts over a week. Each nurse has a skill score and can work multiple shifts (one row per nurse-shift combination). The hospital wants to select which nurses to keep (one decision per nurse, not per shift), limit total hours across all shifts to 100, and maximize total skill score. This demonstrates table-scoped variables — using Table.variable syntax to create one variable per unique entity (nurse) rather than per row (nurse-shift pair). This dramatically reduces solver variables when tables are joined.

SAMPLE DATA
-- Nurses with skill scores
CREATE TABLE Nurses (
    nurse_id INTEGER, name VARCHAR, skill_score INTEGER
);
INSERT INTO Nurses VALUES
    (1, 'Alice', 95),
    (2, 'Bob', 80),
    (3, 'Carol', 90),
    (4, 'David', 70),
    (5, 'Eve', 85);

-- Shifts requiring coverage
CREATE TABLE Shifts (
    shift_id INTEGER, nurse_id INTEGER, shift_date DATE,
    hours INTEGER
);
INSERT INTO Shifts VALUES
    (1, 1, '2025-01-01', 8),
    (2, 1, '2025-01-02', 8),
    (3, 2, '2025-01-01', 6),
    (4, 2, '2025-01-03', 6),
    (5, 3, '2025-01-01', 8),
    (6, 3, '2025-01-02', 8),
    (7, 3, '2025-01-03', 8),
    (8, 4, '2025-01-02', 4),
    (9, 5, '2025-01-01', 8),
    (10, 5, '2025-01-03', 8);
THE QUERY
SELECT n.nurse_id, n.name, n.skill_score, s.shift_date, s.hours,
       n.keep as selected
FROM Nurses n
JOIN Shifts s ON n.nurse_id = s.nurse_id
DECIDE n.keep IS BOOLEAN
SUCH THAT
    SUM(n.keep * s.hours) <= 100
MAXIMIZE SUM(n.keep * n.skill_score);
QUERY BREAKDOWN
1
DECIDE n.keep IS BOOLEAN — Table-scoped variable: the n. prefix means one variable per nurse, not per nurse-shift row. Without n., we'd have 10 variables (one per row); with n., we have only 5 (one per nurse).
2
SUM(n.keep * s.hours) <= 100 — Total hours from selected nurses cannot exceed 100. Since n.keep is table-scoped, it multiplies with each shift's hours — if Alice is kept (keep=1), all her shifts contribute to the total.
3
MAXIMIZE SUM(n.keep * n.skill_score) — Maximize total skill score of selected nurses. Again, the table-scoped variable means each nurse's skill score is counted once per row they appear in — but the solver ensures all rows for the same nurse have the same value.

INTERPRETATION
The solver selected 4 nurses (Alice, Bob, Carol, Eve) with total hours 16+12+24+16 = 68 (under the 100-hour limit). Total skill score: 95+80+90+85 = 350. David was not selected because his skill (70) is low relative to his hours (4), and the solver preferred higher-skill nurses. Notice how each nurse's selected value is consistent across all their shift rows — that's the table-scoped variable in action. Without table-scoped variables, we'd have 10 separate decisions, making the problem harder to solve.
