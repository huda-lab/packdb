Conditional Constraints

Intermediate
WHEN Clause Conditional Constraints Category-Specific Limits
THE PROBLEM

A procurement manager selects items from multiple categories, but each category has different budget limits. Electronics items have a $500 budget, clothing has $300, and food has $200. Additionally, at least 2 items must be selected from each category. The goal is to maximize total value while respecting these category-specific constraints. This demonstrates the WHEN clause — a postfix conditional that applies constraints only to rows matching a condition.

SAMPLE DATA
CREATE TABLE Items (
    item_id INTEGER, name VARCHAR, category VARCHAR,
    value INTEGER, price INTEGER
);
INSERT INTO Items VALUES
    (1, 'Laptop', 'electronics', 100, 800),
    (2, 'Smartphone', 'electronics', 90, 600),
    (3, 'Headphones', 'electronics', 40, 150),
    (4, 'Tablet', 'electronics', 70, 450),
    (5, 'Jacket', 'clothing', 50, 120),
    (6, 'Shoes', 'clothing', 45, 100),
    (7, 'Hat', 'clothing', 20, 40),
    (8, 'Pants', 'clothing', 35, 80),
    (9, 'Apples', 'food', 30, 20),
    (10, 'Bread', 'food', 15, 5),
    (11, 'Cheese', 'food', 25, 15),
    (12, 'Milk', 'food', 20, 8);
THE QUERY
SELECT item_id, name, category, value, price, x as selected
FROM Items
DECIDE x IS BOOLEAN
SUCH THAT
    -- Category-specific budget constraints using WHEN
    SUM(x * price) <= 500 WHEN category = 'electronics' AND
    SUM(x * price) <= 300 WHEN category = 'clothing' AND
    SUM(x * price) <= 200 WHEN category = 'food' AND
    -- At least 2 items per category
    SUM(x) >= 2 WHEN category = 'electronics' AND
    SUM(x) >= 2 WHEN category = 'clothing' AND
    SUM(x) >= 2 WHEN category = 'food'
MAXIMIZE SUM(x * value);
QUERY BREAKDOWN
1
DECIDE x IS BOOLEAN — Binary decision: select (x=1) or skip (x=0) each item.
2
SUM(x * price) <= 500 WHEN category = 'electronics' — Electronics budget: total price of selected electronics items cannot exceed $500.
3
SUM(x * price) <= 300 WHEN category = 'clothing' — Clothing budget: $300 limit.
4
SUM(x * price) <= 200 WHEN category = 'food' — Food budget: $200 limit.
5
SUM(x) >= 2 WHEN category = '...' — At least 2 items must be selected from each category.
6
MAXIMIZE SUM(x * value) — Maximize total value of selected items.

INTERPRETATION
The solver selected 9 items total (3 electronics, 3 clothing, 4 food). Electronics: Headphones ($150) + Tablet ($450) = $600, but the budget is $500, so it picked lower-value items to stay under budget. Clothing: Jacket ($120) + Shoes ($100) + Hat ($40) = $260 (under $300). Food: All 4 items = $48 (well under $200). Each category meets the minimum 2 items requirement. Total value: 40+70+50+45+20+30+15+25+20 = 315.
