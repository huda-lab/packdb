Multi-Period Planning

Advanced Coming Soon

Plan production across multiple time periods with inventory and complex expressions.

Multiple Variables Complex Expressions Linearized Ratios

Coming Soon: This example uses window functions (LAG) over decision variables to model temporal dependencies across periods. Window functions on DECIDE variables are a planned feature and are not yet available in the current build.

Query

SQL 
SELECT
    period,
    demand,
    production_cost,
    holding_cost,
    produce,
    inventory
FROM Periods
DECIDE produce, inventory
SUCH THAT
    produce IS INTEGER,
    inventory IS INTEGER,
    -- Non-negative inventory
    inventory >= 0,
    -- Meet demand each period (inventory balance)
    inventory = COALESCE(LAG(inventory) OVER (ORDER BY period), 0)
                + produce - demand,
    -- Maximum production capacity per period
    produce <= max_capacity,
    -- Linearized constraint: average inventory <= 20% of total production
    -- Instead of: AVG(inventory) <= 0.2 * SUM(produce) / COUNT(*)
    -- We use: SUM(inventory) <= 0.2 * SUM(produce)
    SUM(inventory) <= 0.2 * SUM(produce)
MINIMIZE SUM(produce * production_cost + inventory * holding_cost);
