Examples / Facility Location

Facility Location

Advanced
Quadratic Objective POWER Function MINIMIZE
THE PROBLEM

A company wants to open 3 warehouses to serve 6 customer locations. Each potential warehouse location has an opening cost, and serving a customer from a warehouse incurs a cost proportional to the squared Euclidean distance (penalizing distant warehouses more heavily). The goal is to select exactly 3 warehouses and assign each customer to the nearest open warehouse, minimizing the total cost (opening costs + squared distance costs). This demonstrates quadratic objectives — the POWER(expr, 2) or expr ** 2 syntax for convex quadratic programming.

SAMPLE DATA
-- Potential warehouse locations with opening costs
CREATE TABLE Warehouses (
    warehouse_id INTEGER, name VARCHAR, x_coord INTEGER,
    y_coord INTEGER, opening_cost INTEGER
);
INSERT INTO Warehouses VALUES
    (1, 'Warehouse A', 2, 3, 500),
    (2, 'Warehouse B', 8, 2, 600),
    (3, 'Warehouse C', 5, 7, 450),
    (4, 'Warehouse D', 1, 8, 550),
    (5, 'Warehouse E', 9, 6, 700);

-- Customer locations with demand
CREATE TABLE Customers (
    customer_id INTEGER, name VARCHAR, x_coord INTEGER,
    y_coord INTEGER, demand INTEGER
);
INSERT INTO Customers VALUES
    (1, 'Customer 1', 3, 4, 100),
    (2, 'Customer 2', 7, 3, 150),
    (3, 'Customer 3', 4, 6, 80),
    (4, 'Customer 4', 6, 8, 120),
    (5, 'Customer 5', 2, 9, 90),
    (6, 'Customer 6', 8, 5, 110);
THE QUERY
SELECT w.warehouse_id, w.name as warehouse_name, w.x_coord, w.y_coord,
       w.opening_cost, c.customer_id, c.name as customer_name,
       c.x_coord as cust_x, c.y_coord as cust_y,
       y as open, z as assigned
FROM Warehouses w
JOIN Customers c ON 1=1
DECIDE y IS BOOLEAN, z IS BOOLEAN
SUCH THAT
    -- z=1 means this warehouse serves this customer
    z <= y AND
    -- Each customer must be served by exactly one warehouse
    SUM(z) PER c.customer_id = 1 AND
    -- Exactly 3 warehouses must be opened
    SUM(y) = 3
MINIMIZE
    SUM(y * w.opening_cost) +
    SUM(z * POWER((w.x_coord - c.x_coord), 2) + POWER((w.y_coord - c.y_coord), 2));
QUERY BREAKDOWN
1
DECIDE y IS BOOLEAN, z IS BOOLEAN — Two binary variables: y=1 means open this warehouse, z=1 means this warehouse serves this customer.
2
z <= y — A warehouse can only serve customers if it's open (z links to y).
3
SUM(z) PER c.customer_id = 1 — Each customer is served by exactly one warehouse (PER groups by customer).
4
SUM(y) = 3 — Exactly 3 warehouses must be opened.
5
SUM(y * opening_cost) — Total warehouse opening cost.
6
SUM(z * POWER((w.x_coord - c.x_coord), 2) + POWER((w.y_coord - c.y_coord), 2)) — Quadratic objective: sum of squared Euclidean distances. Using POWER(expr, 2) for convex QP (minimization).

INTERPRETATION
The solver opened Warehouses A, B, and C (total opening cost: $1,550). The quadratic distance cost heavily penalizes distant assignments, so customers are assigned to the nearest open warehouse. Total distance cost: 18. Grand total cost: $1,568. The quadratic objective (POWER) ensures that very distant assignments are disproportionately costly, leading to more balanced facility placement.
